# 第一部分：项目总体设计

## 1. 项目名称与含义

**FundLens**

> FundLens = Fund + Lens，表示像"镜头"一样观察个人基金、ETF、现金理财等资产，让资产结构、收益表现和集中度一目了然。

---

## 2. 项目定位

FundLens 是一个面向个人投资者的**本地化多平台资产快照分析系统**。在竞品格局中，FundLens 位于"Excel 优先 + 季度快照 + 完全本地"这一独特象限——相比松鼠分析账本（移动端、无 Excel 支持）、Portfolio Performance（开源但 Java 桌面端无中文）、天天基金/雪球（平台绑定、需注册）等竞品，FundLens 更侧重**数据可控性、多平台兼容性和季度复盘场景**。

它主要用于：

1. 统一整理支付宝和同花顺中的资产；
2. 基于 Excel 快照数据生成当前资产总览；
3. 分析资产配置结构（含当前 vs 目标配置对比）、持仓收益和产品集中度；
4. 对比基准指数表现，回答"是否跑赢大盘"；
5. 分解收益来源，识别主要收益贡献者；
6. 辅助用户完成季度资产复盘；
7. 保留后续多季度对比和桌面端迁移空间。

**FundLens 明确不做以下事情：**

1. 不自动登录支付宝或同花顺；
2. 不抓取真实账户数据；
3. 不进行自动交易；
4. 不提供买入、卖出、加仓、减仓等投资建议；
5. 不作为专业财务、税务或投资顾问工具；
6. 不上传用户资产数据到任何外部服务器。

---

## 3. 项目背景

用户当前在多个平台持有不同类型资产：

| 平台 | 主要资产 |
|---|---|
| 支付宝 | 纯债基金、固收+、QDII、行业基金、主动基金、余额宝、余利宝、月利宝等 |
| 同花顺 | 场内 ETF，包括宽基 ETF、行业 ETF、主题 ETF、跨境 ETF、债券 ETF、货币 ETF 等 |

**三大核心痛点：**

### 3.1 资产分散

资产分散在支付宝和同花顺等平台中，每个平台只展示本平台资产，用户很难直接看到完整的个人资产全貌。

### 3.2 字段不统一

支付宝中的基金、固收+、余利宝、月利宝等资产，不一定都适合使用"份额、当前价格、成本价"这样的 ETF 字段；同花顺中的 ETF 又更依赖"份额、价格、市值、持仓成本"等字段。因此 FundLens 必须采用更通用的资产快照模型。

### 3.3 复盘效率低

季度资产统计时，用户通常需要反复打开多个平台，手工计算总资产、各平台占比、各类别占比、收益排名、集中度等，效率极低。FundLens 的目标就是将这些重复统计工作自动化和可视化。

---

## 4. 项目目标

FundLens v1.0 的核心目标是：

> 基于用户手动整理或导入的 Excel 资产快照数据，自动保存至本地并生成当前资产总览、资产配置结构、持仓收益表现和产品明细分析结果。

### 4.1 v1.0 核心目标

| 目标 | 说明 |
|---|---|
| 支持 Excel 导入与本地留存 | 用户通过标准模板录入资产，导入后自动保存至本地，支持历史快照切换 |
| 支持多平台统一统计 | 将支付宝、同花顺等平台资产合并展示 |
| 支持多资产类型兼容 | 兼容基金、ETF、现金管理、QDII、固收+等不同资产 |
| 支持当前资产总览 | 展示总资产、总收益、总收益率、产品数量、最大单品占比等核心指标 |
| 支持资产配置分析 | 按平台、资产大类、产品类型、市场区域、资金用途等维度统计 |
| 支持目标配置偏离监控 | 用户设定各资产大类目标比例，系统跟踪当前 vs 目标偏离度，超标时橙色提醒 |
| 支持持仓收益分析 | 展示收益金额、收益率、收益贡献度分解、盈利和亏损排名 |
| 支持基准对比 | 用户设定基准指数（如沪深300），对比总收益率是否跑赢基准 |
| 支持产品明细管理 | 支持筛选、搜索、排序和导出 |
| 支持数据校验 | 对缺失字段、异常金额、收益计算不一致等问题进行三级分层提示 |
| 保持 UI 美观 | 采用简洁金融看板风格，优先满足桌面网页端使用 |
| 本位币统一 | 默认采用人民币（CNY）进行统计展示 |

### 4.2 v1.0 成功标准

第一版的成功不在于功能很多，而在于：

> 用户导入一个 Excel 文件后，能在 1 分钟内看清：当前总资产、各平台资产占比、各类资产占比、当前 vs 目标配置对比、是否跑赢基准指数、持仓收益情况、收益主要来自哪里、占比最高和亏损最明显的产品。

# 第二部分：使用场景与需求分析

## 6. 典型使用场景

### 6.1 场景一：季度资产盘点与历史比对

用户在季度末从支付宝和同花顺查看当前资产情况，并整理为 Excel。随后将 Excel 导入 FundLens，系统自动将文件归档至本地。用户不仅能查看本季度看板，还可以在侧边栏一键切换回历史季度，查看过去的资产分布。

输出结果包括：当前总资产、总持有成本、总持有收益、总收益率、平台资产分布、资产大类分布、产品持仓排名、盈利和亏损产品排名。

### 6.2 场景二：查看当前资产配置

用户希望快速回答：

- 支付宝资产占比多少？同花顺 ETF 占比多少？
- 现金类、固收类、固收增强类、权益类、跨境类资产分别占比多少？
- 权益类资产是否过高？跨境类资产配置是否偏低？
- 单一产品是否占比过高（如超过 20%）？

### 6.3 场景三：查看持仓收益表现

用户希望知道：

- 哪些产品赚钱最多？哪些产品亏损最多？
- 哪些产品收益率最高？哪些产品收益率最低？
- 收益主要来自 ETF、主动基金、QDII 还是债基？
- 是否有部分资产缺少成本或收益数据？

### 6.4 场景四：检查数据质量

用户导入 Excel 后，系统自动提示：

- 哪些产品缺少当前金额（阻断导入）；
- 哪些产品缺少资产分类（警告并自动归类）；
- 哪些产品成本为空，因此不能计算收益率；
- 哪些产品收益金额与"当前金额 - 持有成本"不一致（偏差 > 1 元警告）；
- 哪些产品收益率异常过高或过低；
- 哪些产品代码被 Excel 自动转成数字导致前导 0 丢失。

### 6.5 场景五：对比基准表现

用户想知道自己的投资组合是否跑赢了大盘。系统支持用户设定一个基准指数（如沪深300、中证全债指数），对比总收益率与基准收益率。用户可以从 Excel 中填入基准的期初值和期末值，系统自动计算基准收益率并展示对比图。

### 6.6 场景六：监控目标配置偏离

用户设定了理想的资产配置比例（如现金10%、固收40%、权益40%、跨境10%），希望在每个季度复盘时快速看到当前配置与目标的差距。系统自动对比，当某大类偏离超过 ±5% 时给出橙色提醒，帮助用户决定是否需要在下一季度调整。

## 8. 关键设计原则

### 原则一：以资产快照为核心

FundLens 不记录每日交易流水，而是记录某一统计日期的资产状态。

```
2026-03-31 资产快照 → 导入生成 2026Q1 看板
2026-06-30 资产快照 → 导入生成 2026Q2 看板
2026-09-30 资产快照 → 导入生成 2026Q3 看板
2026-12-31 资产快照 → 导入生成 2026Q4 看板
```

### 原则二：基础数据与计算数据分离

为减少用户填写负担和数据逻辑冲突，Excel 只收集基础原始数据，所有衍生指标（收益、收益率）均由系统自动计算。

- 只要有**当前金额**，就可以进入资产统计；
- 如果有**持有成本**，系统自动计算收益并进入收益分析；
- 如果有**份额和价格**，作为 ETF 的补充信息验证。

### 原则三：资产兼容与容错性

不同平台的资产属性差异极大，系统不强制要求所有资产都有"份额、净值"等字段。支付宝中的余利宝、月利宝等资产只需填写当前金额即可参与总资产统计。

### 原则四：客观呈现，不做决策代劳

系统只做资产统计和收益分析，可以提示：

- 某产品占比较高（如超过 20%）
- 某类资产占比较高
- 某产品当前亏损较大

但绝不给出"建议买入""建议卖出""建议加仓""建议减仓"等投资建议。

### 原则五：目标驱动配置，而非被动展示

系统不仅展示"当前配置是什么样的"，还要帮助用户回答"当前配置与目标差距多大"。参考雪球三分法、且慢再平衡等竞品思路，FundLens v1.0 引入轻量级的目标配置对比：

- 用户可为每个资产大类设定目标比例（可选，非强制）；
- 系统自动生成当前 vs 目标对比图（子弹图/双色柱状图）；
- 当某大类偏离目标超过 ±5% 时，首页卡片给出橙色提醒；
- 系统只做偏离提示，不给出调仓建议。

# 第三部分：数据模型设计

## 9. 数据设计原则

### 9.1 以资产快照为核心

FundLens v1.0 不记录每日交易流水，而是记录某一个统计日期的资产状态。

### 9.2 当前金额是唯一强制金额字段

只要某条资产记录有"当前金额"，它就可以进入总资产统计。

```
当前金额 = 当前市值 / 当前持有金额 / 账户中显示的当前资产金额
```

### 9.3 成本和收益字段允许为空

处理规则：

- 有当前金额 → 参与资产配置统计；
- 有当前金额 + 持有成本 → 可计算持有收益；
- 有持有收益 + 持有成本 → 可计算收益率；
- 缺少持有成本 → 不参与收益率计算。

### 9.4 资产分类与产品类型分离

为避免分类交叉，FundLens 将资产描述拆成三个独立维度：

| 维度 | 字段 | 示例 |
|---|---|---|
| 资产大类 | asset_class | 现金类、固收类、固收增强类、权益类、跨境类、其他类 |
| 产品类型 | product_type | 纯债基金、固收+、宽基 ETF、行业 ETF、QDII、余利宝 |
| 市场区域 | market_region | 中国内地、港股、美国、全球、其他 |

示例：

```
纳指 ETF：
  资产大类 = 跨境类
  产品类型 = 跨境 ETF
  市场区域 = 美国

债券 ETF：
  资产大类 = 固收类
  产品类型 = 债券 ETF
  市场区域 = 中国内地
```

## 12. 收益计算规则

收益统计是 FundLens 最容易出错的部分，必须明确计算口径。

### 12.1 当前金额

```
current_value = 用户在平台中看到的当前持有金额或当前市值
```

所有资产必须填写当前金额。

### 12.2 持有成本

```
cost_amount = 用户累计投入成本或平台显示的持仓成本
```

如果无法获得准确成本，可以为空。

### 12.3 持有收益

计算优先级：

1. 如果用户填写了 `holding_profit`，与系统计算值对比：
   - 若偏差 Δ = |holding_profit - (current_value - cost_amount)| ≤ 1.0 元 → 以用户填写值为准；
   - 若偏差 Δ > 1.0 元且 cost_amount 存在 → 系统发出警告，以系统重新计算的值为准。
2. 如果用户未填写 `holding_profit`，但填写了 `current_value` 和 `cost_amount`，则系统自动计算：

```
holding_profit = current_value - cost_amount
```

3. 如果 `cost_amount` 为空，则 `holding_profit` 可以为空，不强制计算。

### 12.4 持有收益率

计算优先级：

1. 如果用户填写了 `holding_return`，优先使用用户填写值；
2. 如果用户未填写但存在 `holding_profit` 和 `cost_amount`，则系统自动计算：

```
holding_return = holding_profit / cost_amount
```

3. 如果 `cost_amount` 为空或为 0，则收益率为空。

### 12.5 总收益率与收益统计覆盖率

总收益率只基于"有成本数据的资产"计算：

```
总收益率 = 有成本资产的持有收益之和 / 有成本资产的持有成本之和
```

同时在首页显示**收益统计覆盖率**：

```
收益统计覆盖率 = 有成本数据的资产当前金额之和 / 总资产
```

这样可以避免部分资产缺少成本时，总收益率被误解为全资产收益率。

### 12.6 收益统计特殊情况处理

| 情况 | 处理方式 |
|---|---|
| 缺少成本 | 参与总资产和配置统计，不参与收益率统计 |
| 成本为 0 | 不计算收益率 |
| 收益与"当前金额 - 成本"偏差 > 1 元 | 给出警告，系统自动修正 |
| 收益率超过 100% | 标记为异常提示，允许用户确认 |
| 收益率低于 -80% | 标记为异常提示，允许用户确认 |
| 现金类收益 | 可以填 0，也可以留空；留空时不影响资产配置 |

### 12.7 收益率算法说明

> 参考天天基金（摊薄成本法 + 平均成本法）和 Portfolio Performance（TTWR/IRR）的做法，FundLens v1.0 采用**简单收益率**，并在 UI 中透明标注。

FundLens v1.0 计算的收益率类型为**简单持有期收益率**：

```
简单收益率 = (当前金额 - 持有成本) / 持有成本 × 100%
```

**与专业收益率的区别**：

| 收益率类型 | 说明 | FundLens v1.0 | 后续版本 |
|---|---|---|---|
| 简单收益率 | 不考虑资金投入时间差异 | ✅ 当前使用 | — |
| 时间加权收益率 (TTWR) | 剔除现金流影响，衡量投资能力 | ❌ | v1.2+ |
| 内部收益率 (IRR) | 考虑现金流时间价值，衡量资金效率 | ❌ | v1.2+ |

> **UI 标注要求**：在展示收益率的位置旁标注提示文字"当前为简单收益率，不反映资金投入的时间差异"。对于定投场景，简单收益率可能低估或高估真实回报。

### 12.8 基准对比

> 参考 Sharesight 和天天基金的基准对比功能，v1.0 支持用户自设基准指数进行对比。

用户可在系统设置中配置基准对比数据：

| 配置项 | 说明 | 示例 |
|---|---|---|
| benchmark_name | 基准名称 | 沪深300 |
| benchmark_start_value | 基准期初值 | 3500（对应统计日期的指数点位） |
| benchmark_end_value | 基准期末值 | 3850（当前指数点位） |

系统自动计算基准收益率：

```
基准收益率 = (benchmark_end_value - benchmark_start_value) / benchmark_start_value × 100%
```

在首页和收益分析页展示：
- 总收益率 vs 基准收益率 对比
- 相对收益 = 总收益率 - 基准收益率
- 正值为"跑赢基准"（绿色/红色），负值为"跑输基准"（相反色）

## 17. 收益分析模块

### 17.1 功能说明

分析当前持仓的收益表现，帮助用户识别主要收益来源和亏损来源。

### 17.2 分析内容

| 分析项 | 说明 |
|---|---|
| 总持有收益 | 所有可统计资产的持有收益合计 |
| 总收益率 | 基于有成本资产计算 |
| 收益统计覆盖率 | 显示收益分析覆盖了多少资产 |
| **基准对比** | 总收益率 vs 基准收益率对比，展示相对收益（跑赢/跑输） |
| 平台收益对比 | 支付宝、同花顺分别贡献多少收益（柱状图） |
| 资产大类收益对比 | 固收类、权益类、跨境类等分别贡献多少收益 |
| **收益贡献度分解（瀑布图）** | 从总收益出发，逐层展示各资产大类的收益贡献，直观看到"总收益主要由谁贡献" |
| 产品盈利 Top 10 | 盈利金额最高的产品（红色柱状图） |
| 产品亏损 Top 10 | 亏损金额最高的产品（绿色柱状图） |
| 收益率 Top 10 | 收益率最高的产品 |
| 亏损率 Top 10 | 收益率最低的产品 |

### 17.2.1 收益贡献度瀑布图

> 参考天天基金"收益贡献分析"和 Sharesight"Contribution Analysis Report"的设计思路。

瀑布图 (Waterfall Chart) 展示路径：

```
起始：总收益 = 0（基线）
  ↓ +固收类收益（如 +500）
  ↓ +固收增强类收益（如 +100）
  ↓ +权益类收益（如 +1500）
  ↓ +跨境类收益（如 +1200）
  ↓ +现金类收益（如 +0）
  ↓ -亏损产品（如 -450）
终点：总持有收益（如 +2850）
```

效果：用户一眼看出收益主要来自哪个资产大类、哪个大类在拖后腿。

### 17.2.2 基准对比

在收益分析页展示：

1. 总收益率 vs 基准收益率 双柱对比图；
2. 相对收益 = 总收益率 - 基准收益率；
3. 如果用户导入了多个季度快照，可展示"连续 N 个季度跑赢/跑输基准"的趋势。

> **注意**：如果用户未配置基准数据，基准对比图表区域显示引导文案"前往【设置】配置基准指数，查看投资表现是否跑赢大盘"。

### 17.3 缺失数据处理

| 情况 | 处理方式 |
|---|---|
| 缺少成本 | 不参与收益率排名，但在产品明细中标注"无成本数据" |
| 缺少收益 | 若有成本则自动计算，否则标记为"无收益数据" |
| 成本为 0 | 不计算收益率 |
| 当前金额为 0 | 保留记录，提示用户确认 |

| 持有成本不能为负 | 🔴 阻断错误 |
| 产品名称不能为空 | 🔴 阻断错误 |
| 平台不能为空 | 🔴 阻断错误 |

| 资产大类为空 | 🟡 警告（自动归为其他类） |
| 产品类型为空 | 🟡 警告（自动归为其他资产） |
| 持有收益偏差 Δ > 1 元 | 🟡 警告（系统自动修正） |

| 产品代码为数字格式（前导 0 丢失） | 🟡 警告 |
| 重复记录（日期+平台+产品名称相同） | 🟡 警告 |

| 校验规则 | 等级 |

## 23. 技术栈

| 技术 | 用途 |
|---|---|
| Python 3.10+ | 核心开发语言 |

1. 适合快速构建数据分析类网页应用；

3. 开发成本低，可直接本地运行；
4. 非常适合个人工具和 vibe coding 快速迭代；
5. 后续可迁移到更完整的 Web 或桌面应用。

# 第七部分：核心统计指标

## 26. 资产类指标

### 26.1 总资产

```
总资产 = Σ(current_value)
```

### 26.2 平台资产占比

```
平台资产占比 = 某平台资产金额 / 总资产 × 100%
```

### 26.3 资产大类占比

```
资产大类占比 = 某资产大类金额 / 总资产 × 100%
```

### 26.4 产品类型占比

```
产品类型占比 = 某产品类型资产金额 / 总资产 × 100%
```

### 26.5 市场区域占比

```
市场区域占比 = 某市场区域资产金额 / 总资产 × 100%
```

### 26.6 单产品占比

```
单产品占比 = 某产品当前金额 / 总资产 × 100%
```

### 26.7 最大单品占比

```
最大单品占比 = max(单产品当前金额 / 总资产 × 100%)
```

> 该指标用于衡量资产集中度。超过 20% 时首页卡片给出橙色警示。

---

## 27. 收益类指标

### 27.1 总持有成本

```
总持有成本 = Σ(cost_amount)  （仅统计 cost_amount 非空且不为 0 的资产）
```

### 27.2 总持有收益

```
总持有收益 = Σ(holding_profit)  （仅统计有成本数据的资产）
```

其中 `holding_profit` 优先使用用户填写值（偏差 ≤ 1 元时），否则由系统根据当前金额和成本自动计算。

### 27.3 总收益率

```
总收益率 = 有成本资产持有收益之和 / 有成本资产持有成本之和 × 100%
```

> 若总持有成本为 0，则该指标不进行计算与展示。

### 27.4 单产品收益率

```
单产品收益率 = holding_profit / cost_amount × 100%
```

### 27.5 收益统计覆盖率

```
收益统计覆盖率 = 有成本资产当前金额之和 / 总资产 × 100%
```

### 27.6 平台收益贡献

```
平台收益贡献 = 某平台持有收益之和
```

### 27.7 资产大类收益贡献

```
资产大类收益贡献 = 某资产大类持有收益之和
```

### 27.8 收益贡献度

> 参考天天基金"收益贡献"和 Sharesight"Contribution Analysis"设计。

```
单产品收益贡献度 = 该产品持有收益 / 总持有收益（保留正负号）
资产大类收益贡献度 = 该大类持有收益 / 总持有收益（保留正负号）
```

### 27.9 基准收益率

```
基准收益率 = (benchmark_end_value - benchmark_start_value) / benchmark_start_value × 100%
```

### 27.10 相对收益（跑赢/跑输基准）

```
相对收益 = 总收益率 - 基准收益率
```

> 正值为"跑赢基准"，负值为"跑输基准"。

### 27.11 目标配置偏离度

```
资产大类偏离度 = 当前占比 - 目标占比
偏离超过 ±5% 的资产大类数量
```

### 27.12 年化管理费估算

```
年化管理费 = 某产品 current_value × annual_fee_rate
总年化管理费 = Σ(年化管理费)
管理费占收益比 = 总年化管理费 / 总持有收益（仅当总持有收益为正时计算）
```

> 参考 Empower 的费用分析功能，帮助用户意识到管理费对长期收益的侵蚀。

5. 单产品仓位变化；

7. 管理费累计侵蚀分析（P1 升为 v1.1 重点）。

## 31. v1.2：高级分析功能

1. **多算法收益率**：增加时间加权收益率 (TTWR) 和内部收益率 (IRR) 选项（参考 Portfolio Performance、天天基金）；
2. **再平衡自动提醒**：基于用户设定的目标比例，季度末自动检查偏离并提醒（偏离超 ±5% 的 v1.0 版本已做，v1.2 增强为主动推送）；
3. **回撤分析**：多季度快照积累后可计算最大回撤（参考 Portfolio Performance）；
4. **穿透式持仓（自动版）**：通过基金代码自动获取底层持仓数据（v1.0 仅支持手动填写）。

## 32. v1.3：本地数据库与备份

增加 SQLite 本地数据库，用于：

1. 保存历史导入记录；
2. 保存清洗后的资产快照；
3. 保存分类映射表和自定义标签；
4. 保存设置（基准配置、目标配置、图表偏好）；
5. 支持数据备份和恢复；
6. 提高大批量数据下的读取速度。

## 33. v2.0：桌面软件版本 + AI 辅助

1. **桌面端打包**：Python + PyQt 或 Electron，打包为 Windows 应用（.exe）；
2. **AI 辅助分析**：集成本地 LLM（如 Ollama），对资产配置进行中性解读，但不给出投资建议（参考 Moning AI 教练、Wealthfolio）；
3. **FIRE 追踪**：财富自由进度追踪（参考 Doremi）；
4. **自动净值更新**：从公开数据源获取基金净值（参考松鼠分析账本）。

## 34. 远期探索

1. **风险测评问卷**：首次使用时引导用户确定合适的资产配置（参考雪球、且慢）；

3. **股息/分红跟踪**：分红日历和未来收入预测（参考 Sharesight、Moning）；
4. **社区组合分享**：匿名分享资产配置结构（脱敏后），互相参考（参考 Moning、雪球）。
